CLIP Never Forgets (Class-Incremental Learning Baseline)"Frozen VLM Feature는 그 자체로 강력한 클래스 증분 학습 Baseline이다"본 프로젝트는 이미지-텍스트 공동 임베딩 공간을 가진 Frozen CLIP (ViT-B/32) 모델을 활용하여 CIFAR-100 데이터셋 기반의 **클래스 증분 학습(Class-Incremental Learning, CIL)**을 수행하고, 세 가지 분류 방식의 성능 및 Catastrophic Forgetting(파괴적 망각) 현상의 실체를 분석합니다.📌 핵심 요약 (TL;DR)Frozen representation은 과거의 지식을 잊지 않습니다. CIL 과정에서 발생하는 성능 붕괴(Forgetting처럼 보이는 현상)는 유연한 표현(Representation)의 문제가 아니라, 분류기 헤드(Classifier Head)가 최신 테스크의 데이터에만 편향되는 Recency Bias(최신성 편향)로 인해 발생합니다.🛠️ 실험 환경 및 의존성 (Environment & Setup)본 실험은 Google Colab (T4 GPU) 환경에서 구현 및 테스트되었습니다.Bashpip install ftfy regex tqdm

📊 증분 학습 프로토콜 (CIL Protocol)Dataset: CIFAR-100 (클래스 순서는 고정된 시드로 무작위 셔플)Task 구성: 총 10개의 Task $\times$ 각 Task당 10개의 클래스 (총 100개 클래스)Few-shot 환경: 각 클래스당 오직 16장(16-shot)의 이미지 임베딩만 학습용 프로토타입 생성 및 선형 분류기 학습에 사용합니다.평가 방식 (100-way): 평가는 항상 현재 Task 내부가 아니라, 지금까지 등장한 모든 클래스 후보(최종 Task 시점에서는 100-way)를 대상으로 분류 성능을 계산합니다.🔍 비교 방법론 (Methods)세 분류 방식 모두 학습이 불가능하게 고정된 Frozen CLIP ViT-B/32의 이미지 백본을 동일하게 공유합니다.Zero-shot CLIP: 텍스트 프롬프트("a photo of a {class_name}") 기반 분류기 기법으로, 이미지 학습이 전혀 없습니다.Prototype (NCM, Nearest Class Mean): 16-shot 이미지 임베딩의 클래스별 평균(Prototype) 벡터를 구해 코사인 유사도로 분류하며, 분류기 헤드의 갱신(Fine-tuning)이 없습니다.Sequential Linear Probe: 단일 Linear Head를 두고, 새로운 Task가 들어올 때마다 과거 데이터 재학습(Replay) 없이 현재 Task의 데이터로만 연속해서 파인튜닝을 진행합니다.📈 실험 결과 (Results)10개의 Task를 모두 수행한 후 최종 100개 클래스에 대한 평균 정확도(Avg. Accuracy)와 망각 지표(Forgetting Measure)의 결과는 다음과 같습니다.MethodFinal Avg.
